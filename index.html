Lesson 1: The Basics of the Web and HTML

World Wide Web

The World Wide Web (WWW) consists basicaly of Hyper Text Markup Language (HTML) documents linked together by hyperlinks. The WWW was invented in the early 1990s, nowadays there are about 30 billion web pages online on the web. All type of files can be found on the web, besides HTML documents (that hold everything together), there are mainly videos, photos and music files online.

Major components of the web

The major pieces of the web are: (1) client computers with web browsers, (2) the "cloud" and (3) servers that host files. Browsers make requests via the internet to servers using the HTTP protocol (Hypter Text Transfer Protocoll). The servers respond by files, transfered through the internet and displayed by browsers.

HTML

The heart of the web are HTML documents that hold everything together. HTML documents consists of text content, markups (how the content is formated or arranged) and of links (references to other pages or files, e.g multimedia files. Markups are made up of tags, normally in the following structure: <opening tag> content </closing tag>. Tags can also be nested, where the content is another tag inside the tag. There are also void tags that only need opening tags but no closing tags.

These are the most used basic HTML tags:

bold tag: <b> content </b> makes tagged content bold
emphasis tag: <em> content </em> makes tagged content italic
link tag: <a href="wwww.udacity.com"> content</a> which is an anchor tag, using the href attribute containing the url to link to.
image tag: <img src="www.xyz.com/123.jpeg" alt="alternate text">
line break tag: <br> adds a line break (it's a void tag)
paragraph tag: <p>content </p> adds an invisible box around the paragraphed text

Note about white space in HTML: mltiple white space within text is turned into one single white space.

In HTML, there are different type of elements. Inline elements are used within a specific line of content, e.g. the <br> tag. Block elements as the paragraph use an invisible box to define space around it. One other pair of an expample is the inline element of "<span> </span>" opposed to the block element "<div> </div>

When forgetting to set a closing tag, then everything followed by the opening tag is formated by this opening tag. Programmers often say that computers are stupid in the sense that computers only understand clear and error-less instructions (commands) and react very sensitive to typing errors...

The structure of an HTML document is as follows:

<!DOCTYPE HTML>
<HTML>
<HEAD>
<BODY>
tagged content
</BODY>
</HEAD>
</HTML>


Lesson 2: Creating a Structured Document with HTML

Rectangular elements or boxes

All visible elements of a web page are rectangular, although some elements are invisible (like the 'head' element). Even round forms like circles are in fact rectangular elements (with round edges). The rectancular structure of all the elements of a web page allows to organize content in boxes and easily placing and restructuring content. In structuring a web page it is important to start with the large boxes (the main sections of a page) moving to smaller sized boxes containing different content elements like headlines, images, texts etc.)

HTML, CSS and DOM

HTML and CSS are both "languages". HTML controls the "structure" of a web page. CSS controls the "style" of a page (how it looks). By separating format from content, CSS files help implementing one of the most important principles in programming: avoiding repetition or duplication.

HTML has a tree-like structure with nested tags similar to bigger and smaller branches of a tree, defined by the "DOM" (Document Object Model). When programmers talk about the DOM they are talking about the tree-like structure of a page.


Lesson 3: Adding CSS style and HTML structure

CSS: Cascading and Inheritance

With CSS, cascading style sheets, you can define styles that are applied to every element defined, so that you can avoid repetition. Styles can be defined in different places and are applied in the following order, with definitions further down the list overwriting previous definitions:

  • the default style of a browser (different browsers have slightly different styles)
  • stylesheet in a separate file (this is what you will be mostly using
  • stylesheet inside HTML (this can be done for small projects but is not ideal)
  • inline style in an element (this can also be done but should be avoided)

"Cascading" means you keep going down into the more and more specific rules until you find the one that most specifically describes the elements. Rules are applied not only to the elements they directly match, but also to all of those elements' child elements. When rules for elements are defined on more than one level, the most specific rule is applied to every element.

Inheritance is a key feature in CSS, a mechanism by which definitions are applied not only to a specific element ("the ancestor"), but also to its nested elements ("the descendants"). Descendant elements may inherit CSS property values from any ancestor element enclosing them.

CSS definition

In CSS you first write a so called "selector", for example a html tag like body or h1 or by defining a class (".class_name". This defines which elements of the page a specific style should apply to. In general it is good to use so called "semantic tags" like h1, h2 etc., pre-defined html tags available for structuring documents in order to define styles for specific types of elements (like h1).

With CSS, style attributes are attached to specific elements. All CSS style attributes can be found in this CSS reference.

There are 3 ways how to include CSS styling in a web page:

  • Method 1: write CSS in the <head> of your HTML
  • Method 2: Link your HTML to a separate CSS file
    • (like done with this litte web page...)
  • Method 3: Write your style inline with your HTML
Method 1 is only suitable for very short projects. Method 2 is the best method since it allows to really seperate style definition from the functional code. Method 3 is not reccomended since it leads to a lot of repetition and duplicate code.

Positioning content using the Box Model

Content on a web page is positioned using the box model. A web page consists of different boxes. A box is defined by 4 nested areas or components; for every of these 4 components the size or width can be defined:

  • content
  • padding
  • border
  • margin
A sample page displaying the box model can be found here. There are two techniques you can use to help deal with sizing issues of boxes. (1) Set sizes in terms of percentages rather than pixels. (2) Set the box-sizing attribute to border-box for every element.

Code, test and refine

Software development follows these three steps over and over again:

  1. code
  2. test
  3. refine
In iteration after iteration, the programmer codes a new element, then instantly tests it against the design mockup and then refines it, until it works and displays as desired or defined by the design mockup.

But how to get to the first code from your design mockup? You should follow these steps:

  1. Look for natural boxes by visually boxify the design mockup to get the structure of the page
  2. Look for repeaded styles and semantic elements: identify headings, sub-headings, text content etc. by assigning tags like h1, h2 p etc.
  3. Write HTML code
  4. Define the styles from biggest to smallest boxes. Define the size of the boxes (by sizing content, padding, border and margin) and position the boxes on the screen

It is very important to verify the correctness of the produced HTML and CSS code. You can do that using the these validators:

  • To verify HTML: http://validator.w3.org/#validate_by_input
  • To verify CSS: http://jigsaw.w3.org/css-validator/#validate_by_input
